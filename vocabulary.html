Turing Front-End Engineering

Vocabulary


Being familiar with key terms and concepts is a big help in your early days at Turing, so we've put together some vocabulary that will help you be able to 'speak code'.

  1. API

    We're building programs that can talk to other programs. Their APIs are their languages. Sort of.

  2. WYSIWYG

    WYSIWYG stands for "What You See Is What You Get." A WYSIWYG editor allows you to visually see the results as you're editing. A good example of this is the editor in Gmail and other web-based email clients.

  3. IP Address

    Every server on the web has a unique IP address, which is like a phone number. "63.145.15.99" is an example of an IP address.

  4. Domain

    Trying to remember the unique IP address for every website on the Internet isn't really a long-term game. Almost all websites have a domain name (e.g. "www.facebook.com") that redirects to their IP address.

  5. Web Server

    A web server is a program that listens for HTTP requests, handles them, and sends responses back to the browser that made the request.

  6. Bugs

    A bug is some kind of error or issue in our program. Back in the days of punch card computers, Grace Hopper found a literal bug in the computer that was causing erroneous results.

  7. The Cloud

    "The Cloud" is the collective name for a collection of computers on the Internet where you might store data or pictures of your cat.

  8. Command Line

    The command-line is a text-based interface for working with your computer. You can do a whole bunch of chill stuff that there isn't a graphical user interface for.

  9. Version Control

    Version control—most notably, git—allows you to save different versions of your application as you go along. You can even split off for a bit

  10. Grid Systems

    Just because we're not designers doesn't mean we shouldn't appreciate and respect their grids.

  11. Cookies

    There's probably a pun we could come up with, but we take security seriously.

  12. 404 Error

    The web's best easter egg.

  13. Responsive Design

    Keep your mind like water, but your design should be fluid and screen independent.

  14. SEO

    Despite what you might read, it's basically just black magic, the dark arts, the voodoo that gives traffic.

  15. Hex

    Because green to you might not be green to me. So there's these really specific codes we use for colors.

  16. CMS

    All applications are basically just content management systems anyway.

    First learn to express yourself. Then learn to code.
